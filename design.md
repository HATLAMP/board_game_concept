# Design for website workflow

## User Stories
### Assuming version of game where you commit each move.
Play a simple game against a basic bot.
- A Player wants to see their pieces.
- A Player wants to select their pieces.
- A Player wants default pieces they can select.
- A Player wants to be able to customise piece health, energy and attack.
- A Player wants to see a board.
- A Player wants to place pieces on the board.
- A Player wants to commit to start game.
- A Player wants to be prompted to move a piece.
- Select movement, commit move.
- If piece comes into contact with other piece combat starts.
- Piece destroyed if killed.
- Win if other players pieces all run out of energy or has no more pieces left vice versa.
- Sees endscreen with outcome.
- Taken back to first screen and prompted to start again.
